1. Gitlab

  • 実施内容
    1-1. 初期設定
    1-2. カタログ修正確認
    1-3. カタログプロジェクトの作成/保存

1-1. 初期設定

  1. Gitlabにアクセスし、“root"のパスワードを設定し、管理者権限でログイン

  2. 「Admin Area」にアクセス

  3. 「Settings」-> 「CI/CD」を選択

  4. 以下のチェックを外す

    • 「Default to Auto DevOps pipeline for all projects」
    • 「Enable shared runners for new projects」

  5. 「Settings」-> 「General」を選択

  6. 「Sign-up restrictions」にある「Sign-up enabled」のチェックを外す

1-2. カタログの修正確認

保存するカタログがお客様向けになっているか確認
“各チャート"下にある”README.md"values.ymal"と”question.yaml”を編集

  • README.md
    • 使用するDockerイメージの確認の記述を変更（cleansing,learning)
  • values.yaml
    • PROXYのURL
    • repository,tagの名前
    • system.internalのIPアドレス
  • question.yml
    • variable: defaultImage
    • ”description”と”default”、”tag”の記述
    • 各タグの”default”の内容

1-3. カタログプロジェクトの作成/保存

管理者権限でカタログ保存するためのプロジェクトを作成

  1. 「New Project」を選択し、 以下の内容で設定

    • プロジェクト名：custom_catalog
    • Visibility Level：”Public”

  2. 「Create Project」を選択

  3. 以下の手順でカタログを保存する

    $ git config --global user.name "Administrator"
$ git config --global user.email "admin@example.com"
$ cd <custom_catalog dir>
$ git init
$ git remote add origin http://xxx.xxx.xxx:10080/root/custom_catalog.git
$ git add .
$ git commit -m "Initial commit"
$ git push -u origin master

3-1. ユーザ登録/権限登録

  1. Rancherにアクセスし、IDを"admin”、PWを”password”で管理者としてログイン

  2. クラスター管理画面にある”ユーザ”を選択し、”ユーザを追加”を押す

  3. ユーザー名、パスワード、表示名は下記の通り設定し、”” User-Baseと”Use Catalog Templates”にチェック

3-2. カタログ登録

  1. 作成されたプロジェクトの”App”を選択

  2. “Managed Catalog"から”カタログ追加”を選択

  3. 以下のように設定し、作成ボタンを押す

    • URL: http://xxx.xxx.xxx:10080/root/custom-catalog.git
    • ブランチ：master

  4. ”起動”を押してカタログの表示を確認（無い場合は”読み込み”ボタンを押してみる）

    io.rancher.rke.container.name kubelet
    io.rancher.rke.container.name kube-proxy
    io.kubernetes.pod.namespace monitoring
    io.rancher.rke.container.name share-mnt
    io.rancher.rke.container.name nginx-proxy
    io.rancher.rke.container.name service-sidekick

5. Grafana

  • 実施内容
    5-1. 可視化ツールのデプロイ 5-2. 初期設定 5-3. ユーザ登録

5-1. 可視化ツールのデプロイ

“monitor/manifest.yaml"にあるファイルをデプロイ

$ kubectl apply -f manifest.yaml

5-2. 初期設定

  1. Grafana（http://xxx.xxx.xxx.xxx:30300）にアクセスし、IDを"admin”、PWを"admin"でログイン

  2. パスワードの変更を求められるので"password"で設定

  3. データソースの追加

    • 以下の内容で設定し、“Save&Test"を実行
    • Time series databases：Prometheus
    • Name: gpu_cluster
    • URL: 管理サーバのURL:9090
    • Skip TLS Verify: チェックを入れる

  4. GPUのメトリックスをみるためにグラフを追加
    左タブの”＋”から”import”を選択し、予めコピーしていたテンプレートを貼り付ける

5-3. ユーザ登録

  1. “Server Admin"から”Users”を選択

  2. ”New Users"を押して、以下のようにユーザを登録

    Name Username 　Password 　Email
    Moritaka Oota m-oota Password123 m.oota.nm@hitachi-kenki.com
    Satomi Kunimatsu s-kunimatsu Password123 s.kunimatsu.ja@hitachi-kenki.com
    Roh Joonyoung j-noh Password123 j.roh.qy@hitachi-kenki.com

  3. ”Configuration”から”Users”を選択 　

  4. 追加したユーザのRoleが”Viewer”になっていることを確認　　

6. その他

Dockerイメージの定期削除

  1. ~/image/clean.shに実行権限を付与

    $ chmod +x ~/image/clean.sh

  2. /etc/cron.d/cleanを644のパーミッションで作成

    SHELL=/bin/sh
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
# m h dom mon dow user command
00 3 1 * * root /bin/bash /home/<User>/image/clean.sh >> /home/<User>/image/result.log

  3. cronの再起動

    sudo service cron restart
