Kubernetes導入

作業内容

Rancherを利用してKubernetes環境を立ち上げる

  1. Rancherにログイン
  2. クラスターの初期設定と構築
  3. Kubectlへのconfig登録
  4. NVIDIA Device pluginの適応

1. Rancherにログイン

1-1. 管理サーバで動作しているRancherにアクセス
　http://<管理サーバのIP>:9080

1-2. 管理者向けパスワードを登録
　パスワード：password

2. クラスターの初期設定

2-1.「クラスター追加」ボタンを押す
2-2.「カスタム」ボタンを押す
2-3. 以下の様に設定を行い、「次へ」を押す

  • クラスター名：dgx-cluster
  • Kubernetes バージョン：v1.17.x-rancher1-1
  • ネットワークプロバイダー：Canal
  • イングレスプロバイダー：無効
  • Metrics Server Monitoring：無効
  • ノードの Docker バージョン：サポートされている Docker バージョンを要求
  • Recurring etcd Snapshot Enabled：無効

2-4. ノードオプションで管理サーバとDGX向けのコマンドを生成と実行

  • 管理サーバ

    • ノードロール：「etcd」「control」「worker」を選択
    • ノードアドレス：管理サーバのIP(「パブリック IP」と「プライベート IP」の両方)
    • ノード名：management

  • DGX向け

    • ノードロール：「worker」のみ選択
    • ノードアドレス：DGXのIP(「パブリック IP」と「プライベート IP」の両方)
    • ノード名：dgx-station-01

Kubernetes導入のためのコマンドを実行を各サーバで実行し、クラスター管理画面で状態が「Active」になるまで待つ

2-5. 各ノードにラベル付け

  • 管理サーバ

    • exporter=master

  • DGX

    • gpu=dgx-s
    • hardware-type=NVIDIAGPU

3. Kubectlへのconfig登録

3-1. クラスターを開き、kubeconfigファイルをコピー
3-2. 管理サーバにコピーしたkubeconfig情報を"~/.kube/config"に保存
3-3. kubectlを実行し、情報が取得できればOK

$ kubectl get nodes

4. NVIDIA Device pluginの適応

4-1. 以下のコマンドを実行

$ cd
$ kubectl apply -f nvidia-device-plugin.yml

4-2. プラグインの動作確認のために以下を実行し、GPUの情報が取れればOK

$ kubectl run gpu-test --rm -t -i --restart=Never --image=nvidia/cuda:10.0-base --limits=nvidia.com/gpu=1 -- nvidia-smi
